# ADR-0006: 本文項目をマインドマップツリーに統合

- ステータス: 承認済み
- 確信度: 高（コミットメッセージに「下部ボディパネルを廃止しツリー表示に一本化」と明示）
- 日付: v1.7.0（commit 28f1dfe）

## コンテキスト

Markdown 見出しの下に書かれたリスト項目（`- [ ] text` / `- [x] text` / `- text`）を、マインドマップ上でどう見せるかという設計課題があった。v1.7.0 以前は「下部ボディパネル」として見出しノードとは別区画で表示していた。

## 決定

下部ボディパネルを廃止し、本文リスト項目を見出しノードの右横に「本文ノード」として描画することにした。視覚的区別として：
- **見出しノード**: 実線ボーダー
- **本文ノード**: 破線ボーダー＋半透明背景
- **接続線**: 見出し間=実線、本文=破線グレー

## 理由

コミットメッセージより「下部ボディパネルを廃止しツリー表示に一本化」。ツリーとパネルで情報が分断されていたユーザー体験の問題を解消し、見出し構造と本文項目の関係をひとつの視覚空間で把握できるようになった。

## 捨てた選択肢

- **下部ボディパネル方式**: v1.6.x 以前の方式。ツリーとパネルを分けて見る必要があり、空間的な関連性が見えにくかった。
- **本文は表示しない**: 見出しの内容が失われてマインドマップとして情報量が不足。

(要確認) 下部パネル時代の具体的なユーザーフィードバックや問題点についての記録は git log に見当たらなかった。
