# ADR-0001: ファイル名を常にルートノードとする

- ステータス: 承認済み
- 確信度: 高（コミットメッセージに明示）
- 日付: v1.4.x 期（commit 7c027ef）

## コンテキスト

初期実装では、Markdownファイルの H1 見出しをルートノードとする設計があり、H1 の有無によって条件分岐が必要だった。この分岐はパーサーとシリアライザーの両方に複雑さをもたらしていた。

## 決定

ファイル名（拡張子なし）を常に level-0 のルートノードとし、`#`〜`######` の見出しはすべてその子ノードとして扱う。H1 をルートとする条件分岐を廃止した。

## 理由

コミットメッセージ「シンプルで一貫した挙動になる」より。H1 の有無に関わらず挙動が同一になることで、パーサーの分岐が削除でき、シリアライザーも「level 0 は見出しマーカーを付けない」という単純な規則に統一できた（`markdownParser.ts:75-76`）。

## 捨てた選択肢

- **H1 をルートノードとする方式**: H1 が存在するファイルとしないファイルで挙動が異なり、かつ H1 を持たないファイルへの対処が必要だった。コミット 7c027ef の変更前まで存在した方式。
